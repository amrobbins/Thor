# Setup
############################

Use Ubunut 24.04 - Later maybe ok, earlier then you need to get python3.12 installed. Tested using Ubuntu 24.04.

Sanity check nvidia driver and cuda

nvidia-smi

nvcc --version

Ensure both work and that you have cuda 13 installed.

sudo apt update
sudo apt-get -y install cudnn9-cuda-13

sudo apt install -y \
    build-essential \
    cmake \
    ninja-build \
    git \
    pkg-config \
    libomp-dev \
    ccache \
    clang-format \
    libarchive-dev \
    liburing-dev

sudo apt install -y python3 python3-venv python3-dev python3-pip python-is-python3

sudo apt install -y libncurses-dev graphicsmagick libgraphicsmagick++1-dev nlohmann-json3-dev libboost-all-dev

############################
# Building
############################

cd ~
~$ git clone git@github.com:amrobbins/Thor.git
~$ cd ~/Thor
~/Thor$ python3 -m venv .venv
~/Thor$ source .venv/bin/activate
~/Thor$ python -m pip install --upgrade pip
~/Thor$ python -m pip install nanobind yapf isort

~/Thor$ mkdir build
~/Thor$ cd build
~/Thor/build$ cmake .. -G Ninja -DCMAKE_BUILD_TYPE=Debug
~/Thor/build$ cmake .. -G Ninja -DCMAKE_BUILD_TYPE=Release
~/Thor/build$ cmake .. -G Ninja -DCMAKE_BUILD_TYPE=RelWithDebInfo
~/Thor/build$ cmake --build .

# Build the wheel from the repo root directory:
~/Thor$ python -m build --wheel bindings/python


############################
# Testing
############################

# Test all
~/Thor/build$ cmake --build . --target check

# Test all cpp
~/Thor/build$ cmake --build . --target check-cpp

# Test all python
~/Thor/build$ cmake --build . --target check-python

# Run gtests natively so can pass flags
~/Thor/build$ ./thor_tests --gtest_filter=Sgd*

# Run pytest natively so can pass flags
~/Thor/bindings/python$ pytest test/core/smoke_test.py::test_smoke

# CLion Setup
############################

When setting up IDE, clion recommended, point it to the python interpreter at <repo root>/.venv/.
Go to  settings → editor → general → on save and check
    1. remove trailing blank lines
    2. ensure every file ends in newline

Settings → Editor → Code Style → C/C++
Check use clang-format
Settings → Tools → Actions on Save
Check reformat code. Choose C++, and cmake
Settings → Plugins → Marketplace
Install File Watchers

Settings → Tools → File Watchers → + → Custom
Name: YAPF
File type: Python
Scope: Project Files
Program: path to yapf i.e. <content root>/.venv/bin/yapf
Arguments:
--in-place $FilePath$
Working directory: $ProjectFileDir$
Output paths to refresh: $FilePath$
Make sure “Trigger the watcher on save” (or “On Save”) is checked.

The File Watcher will continuously try to format your python code as you are typing, by default, fix that:
Settings / Preferences → Tools → File Watchers
Select your YAPF watcher → Edit
In Advanced Options, UNcheck:
    Auto-save edited files to trigger the watcher
With that unchecked, the watcher runs only when a real save happens (explicit save or when CLion saves on frame deactivation).
Prevent “format loop” / extra jumps:
In that same watcher, if you see it, uncheck:
    Trigger the watcher on external changes (prevents a watcher rewrite from re-triggering itself / causing extra reload-jitter)
